## 详细描述

哔哩哔哩历史记录保存与分析是一款非官方浏览器扩展，用于把哔哩哔哩账号中的历史记录保存到当前浏览器，方便长期查找、按日期筛选、备份和分析。

主要功能：

- 手动保存全部历史记录，并自动检查最近的新记录；
- 按标题、UP 主、BV 号、AV 号、日期和内容类型搜索；
- 显示视频长度、观看位置和观看进度；
- 分析普通视频的平均长度、平均观看进度、跳出概率、看完概率、常看分区和常看 UP 主；
- 提供逐日观看进度、估算观看时长和其他趋势图表；
- 支持 JSON 备份、恢复和 CSV 表格导出；
- 支持时区设置，避免跨时区时记录被划分到错误日期；
- 可选连接用户自行搭建的 HTTPS 云端，实现多设备记录合并。
- 可随时删除云端副本并清除本机保存的云端地址和同步密钥。

隐私说明：历史记录默认仅保存在当前浏览器。本扩展不会删除或修改哔哩哔哩账号中的记录，不包含广告或开发者遥测。云端同步默认关闭，只有用户主动配置自己的云端地址并点击同步时才会传输数据。

本扩展与哔哩哔哩、上海宽娱数码科技有限公司及其关联公司不存在隶属、授权或背书关系。

开源来源说明：本扩展基于现有 MTI/Bilibili 历史记录扩展项目二次开发，当前可确认的上游项目为 Bilibili 无限历史记录 / bilibili-history-wxt，许可证为 MIT。当前版本已调整为历史记录保存与观看分析用途。

## Chrome 隐私表单

单一用途：读取用户在哔哩哔哩账号中的历史记录，将其保存在浏览器本地，并提供搜索、备份、导出和观看分析。用户可选择把记录同步到自己配置的 HTTPS 云端。

`storage` 权限说明：保存用户设置、同步状态和界面偏好。历史记录主体保存在扩展自己的 IndexedDB 中。

`cookies` 权限说明：检查哔哩哔哩登录状态，使扩展能以当前登录用户身份请求官方历史记录接口。Cookie 不会写入数据库、导出文件或云端同步内容。

`alarms` 权限说明：按用户设置的时间间隔唤醒后台任务，检查并保存新的历史记录。

主机权限说明：仅访问 `bilibili.com` 及其子域，用于读取哔哩哔哩官方历史记录接口和登录 Cookie；不申请所有网站访问权限。

远程代码：否。扩展不会下载或执行远程 JavaScript、WebAssembly 或其他可执行代码。网络请求只交换历史记录 JSON 数据。

数据用途声明：数据只用于扩展的保存、查找、备份、导出、恢复、分析和用户主动发起的云端同步功能；不会出售，不会用于广告或信用评估，也不会传输给开发者运营的服务器。

## Firefox 数据声明

默认必需数据传输：无。

可选数据传输：

- `browsingActivity`：用户主动云同步时发送已保存的哔哩哔哩历史记录和观看进度；
- `authenticationInfo`：用户主动云同步时向其配置的云端发送同步密钥以验证请求。

上述数据只发送到用户自行填写的 HTTPS 地址，并在 Firefox 中通过可选数据权限单独征得同意。

用户可以在设置页删除云端副本并停用同步；停用后会清除当前浏览器保存的地址、同步密钥和设备随机标识。

## 审核备注

1. 打开扩展弹窗，点击“打开记录与分析”，可在不提供测试账号的情况下查看页面、设置和导入功能。
2. 读取在线历史记录需要审核人员在自己的哔哩哔哩账号中登录；开发者不提供或收集账号凭据。
3. 云端同步不是核心功能，默认关闭。只有填写地址和密钥并主动点击按钮后才会联网。
4. 云同步测试可不执行；设置页面内已提供完整的 Cloudflare Worker 示例和数据传输说明。
